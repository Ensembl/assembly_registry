Assembly Registry FAQ

What is Assembly Registry?

Assembly Registry is a small service to keep track of existing assemblies and add new ones as they come along.

What services are provided by Assembly Registry?

The web service would provide the following:
- querying of existing assemblies and providers
- adding new assemblies, creating new unique identifiers based on the gender, species and assembly
- querying of existing species from the taxonomy database, using taxon id or species name

What do you mean by GenSpeNum? How is it derived?

GenSpeNum stands for Gender + Species scientific name + Assembly Number.

Unique attribute derived by appending Gender (first char) + Species scientific name (first 3 chars from genus + first 3 chars from species) + Assembly Number (incremented every time if not given)

Example:
female + Astatotilapia calliptera + 2 = fAstCal2
  None + Astatotilapia calliptera + 2 =  AstCal2

How Authentication system works in Assembly Registry?

Assembly Registry supports both session based authentication via the Login interface for web browsable API and token based authentication for REST API.

Authentication workflow (Session Based):
- Registration: Register yourself by filling out a simple signup form.
- Login: Login via the login interface with the user credentials.
- GET, POST, PUT, DELETE operations: After successful authentication, you can do all the operations via the browsable API
Authentication workflow (TokenAuthentication Based):
- Registration: Register yourself by filling out a simple signup form. All successfully registered users will be assigned a unique Authentication Token
- Login: Login using the user credentials to retrieve the Authentication Token via the /accounts/get_auth_token endpoint.
 
curl -X POST --header 'Content-Type: application/json' --header 'Accept: application/json' -d '{ "username":"xxx","password":"xxx" }' '{{hostname}}/accounts/get_auth_token/'
You can also retrieve your Token anytime by clicking on the AuthToken button from the site after logging in. You will receive the token as a json response
{"token":"fee6d25ca6dce45952f323bde6dbebb2f3bc7aff”}
For clients to authenticate, the token key should be included in the Authorization HTTP header. The key should be prefixed by the string literal "Token", with whitespace separating the two strings.

For example:
curl -X DELETE --header 'Authorization: Token fee6d25ca6dce45952f323bde6dbebb2f3bc7aff' '{{hostname}}/assembly/gen_spe_num/mHomSap3/'
Please Note: After creating a assembly, you can only UPDATE or DELETE records created by yourself and not created by any others.

How to do GET, POST, PUT, DELETE operations via REST API?

GET
curl -X GET --header 'Accept: application/json' '{{hostname}}/assembly/'
curl -X GET --header 'Accept: application/json' '{{hostname}}/assembly/?project_name=VGP'
curl -X GET --header 'Accept: application/json' '{{hostname}}/assembly/?gen_spe_num=fAstCal2'
curl -X GET --header 'Accept: application/json' '{{hostname}}/assembly/?assembly_version=38'

Please note: Authentication Token not needed for GET requests.
POST
curl -X POST --header 'Content-Type: application/json' --header 'Accept: application/json' --header 'Authorization: Token fee6d25ca6dce45952f323bde6dbebb2f3bc7aff' -d '{ "taxon_id": "9601", "scientific_name": "Homo sapiens", "project_name": "HHH", "gender": "male", "data_release":"public" }' {{hostname}}/assembly/
PUT (UPDATE)
curl -X PUT --header 'Content-Type: application/json' --header 'Accept: application/json' --header 'Authorization: Token fee6d25ca6dce45952f323bde6dbebb2f3bc7aff' -d '{ "taxon_id": "9601", "scientific_name": "Homo sapiens", "project_name": "ddd", "gender": "male" }' '{{hostname}}/assembly/gen_spe_num/mHomSap3/'
DELETE
curl -X DELETE --header 'Authorization: Token fee6d25ca6dce45952f323bde6dbebb2f3bc7aff' '{{hostname}}/assembly/gen_spe_num/mHomSap3/'

What is the technology stack of Assembly Registry?

Python3, Django, Django REST Framework and Django REST Swagger. Backend support is provided by MySQL.
